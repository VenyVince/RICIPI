## 1. 프로젝트 소개 ##
### 1.1 프로젝트 개요 ###
- 프로젝트 명: RICIPI
- 프로젝트 형태: 개인 프로젝트
- 개발 목적: 실사용 서비스 지향
- 프로젝트 기간: 2025.12.21 ~ 2026.01.07

### 1.2 기획 배경 ###
- 기존 레시피 서비스(만 개의 레시피 등)에서 가지는 한계
    - 직접 레시피를 하나하나 골라야 함.
    - 내 상황과 달리 새로운 재료들을 사와야 함.(별도의 수고)

### 1.3 프로젝트 목표 ###
- 메뉴 선택의 피로↓
- 재료 낭비↓



## 2. 문제 정의 ##
- 매 식사마다 메뉴 선택의 피로함
- 냉장고의 재료가 낭비
- 레시피가 과잉되게 많음.(음식마다 레시피 다수 존재)

## 3. 서비스 개요 ##
### 3.1 핵심 기능 ###
- 남은 재료 기반 활용 레시피 추천
- 메뉴 결정 보조 기능
- 사용자의 취향에 맞춘 레시피 추천

### 3.2 향후 확장 기능 ###
- 알레르기 / 식단 필터
- AI 추천 및 고도화
- 커뮤니티 기능을 통한 레시피 등록/조회
